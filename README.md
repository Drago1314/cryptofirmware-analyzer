# 🔐 CryptoFirmware Analyzer

**AI/ML-Based Identification of Cryptographic Primitives in Multi-Architecture Firmware Binaries**

[![Python](https://img.shields.io/badge/Python-3.8%2B-3776ab?logo=python&logoColor=white)](https://python.org)
[![Streamlit](https://img.shields.io/badge/Streamlit-FF4B4B?logo=streamlit&logoColor=white)](https://streamlit.io)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

CryptoFirmware Analyzer is an interactive tool that scans firmware binaries to detect embedded cryptographic algorithms using a combination of **signature-based constant matching**, **Shannon entropy analysis**, and **Random Forest ML classification**. It supports multiple binary formats and processor architectures out of the box.

---

## ✨ Features

- **🎯 Signature-Based Detection** — Identifies AES S-Box, SHA-256/SHA-1/MD5 init constants, RSA PKCS#8 OIDs, and text markers (PEM headers, library references)
- **🤖 ML Classification** — Random Forest model classifies 256-byte blocks into 7 categories: encrypted data, machine code, plaintext/config, AES constants, hash constants, RSA key material, and compressed data
- **📈 Entropy Profiling** — Full binary entropy graph with high-entropy region highlighting (encrypted/compressed sections)
- **🗺️ 2D Entropy Heatmap** — Binvis-style visual heatmap for rapid structural overview
- **🔬 Hex Dump Viewer** — Formatted hex dump at each detection offset with ASCII decoding
- **🏗️ Multi-Architecture Support** — ELF (x86, x86-64, ARM, AArch64, MIPS, RISC-V, PowerPC), PE (Windows), Mach-O (macOS/iOS), U-Boot images, and raw binaries
- **📊 Binary Comparison** — Upload 2–4 binaries for side-by-side entropy, ML, and detection comparison
- **📥 Report Export** — Download a styled HTML analysis report

## 🚀 Quick Start

### Option 1: One-Click Launch (Windows)

Double-click **`RUN_ME.bat`** — it will:
1. Check for Python
2. Install all dependencies
3. Generate a test firmware binary
4. Launch the app at `http://localhost:8501`

### Option 2: Manual Setup

```bash
# Clone the repository
git clone https://github.com/<your-username>/cryptofirmware-analyzer.git
cd cryptofirmware-analyzer

# Install dependencies
pip install -r requirements.txt

# (Optional) Generate a test binary
python generate_test_binary.py

# Launch the app
streamlit run app.py
```

The app will open automatically at **http://localhost:8501**.

## 🔍 How It Works

### 1. Signature Scanning
The tool searches the binary for known cryptographic constants:
| Algorithm | What It Looks For |
|-----------|-------------------|
| **AES** | S-Box lookup table (256 bytes), OpenSSL/MbedTLS symbol references |
| **SHA-256** | Initial hash values (H0–H7), round constants (K) |
| **SHA-1** | Initialization vector (H0–H4) |
| **MD5** | Initialization constants |
| **RSA** | PKCS#8 OID sequences, PEM headers |
| **X.509/TLS** | Certificate PEM markers |

### 2. Entropy Analysis
Shannon entropy is computed per 256-byte block. Regions with entropy > 7.2 bits/byte strongly indicate encrypted or compressed data, while regions below 5.5 suggest plaintext or structured code.

### 3. ML Classification
A Random Forest classifier (150 trees) is trained on synthetic firmware data covering 7 region types. Each 256-byte block is independently classified with a confidence score.

**Feature vector (267 dimensions):**
- Shannon entropy, byte mean/std/median
- Full 256-bin byte histogram
- Printable ratio, zero-byte ratio, high-byte ratio
- Chi-squared uniformity test
- Maximum run length, bigram entropy, byte range
